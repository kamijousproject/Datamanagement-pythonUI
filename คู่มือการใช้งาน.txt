================================================================================
                    คู่มือการใช้งาน Phone Data Manager
================================================================================

โปรแกรมจัดการข้อมูลเบอร์โทรศัพท์แบบครบวงจร
Database: SQLite (phone_data.db)
พัฒนาโดย: Kamijous Project

================================================================================
สารบัญ
================================================================================
1. การนำเข้าข้อมูล
2. การจัดการข้อมูล
3. การย้ายข้อมูล
4. การส่งออกข้อมูล
5. การรวม 2 ไฟล์
6. การกรองเบอร์ซ้ำ

================================================================================
1. การนำเข้าข้อมูล
================================================================================

ขั้นตอน:
1. เปิดแท็บ "นำเข้าข้อมูล"
2. เลือก Table (phone_data_set_1 ถึง 15)
3. กรอกข้อมูล:
   - ชื่อชุดข้อมูล: เช่น "Lead_Facebook_Nov"
   - วันที่รับข้อมูล: เลือกจาก Calendar
   - แหล่งที่มา: เช่น "Facebook Ads"
   - รายละเอียด: หมายเหตุเพิ่มเติม
   - ประเภท: เช่น "Lead", "Customer"
4. เลือกไฟล์ .txt (เบอร์ 1 บรรทัด 1 เบอร์)
5. ตรวจสอบตัวอย่าง และเบอร์ซ้ำ
6. กดบันทึก

รูปแบบไฟล์:
0812345678
0823456789
0834567890

หมายเหตุ:
- เบอร์ซ้ำในฐานข้อมูลจะไม่ถูกนำเข้าอีก
- เบอร์ซ้ำในไฟล์จะนำเข้าเพียงครั้งเดียว
- ไฟล์ต้องเป็น UTF-8

================================================================================
2. การจัดการข้อมูล
================================================================================

ขั้นตอน:
1. เปิดแท็บ "จัดการข้อมูล"
2. เลือก Table ที่ต้องการดู
3. ใช้ฟิลเตอร์:
   - ค้นหาเบอร์: พิมพ์เบอร์หรือบางส่วน เช่น "081"
   - ชื่อชุดข้อมูล: พิมพ์ชื่อที่ต้องการหา
   - วันที่: เลือกช่วงวันที่
4. กด "ค้นหา"
5. ดูผลลัพธ์ในตาราง
6. ดูสถิติด้านล่าง (จำนวนทั้งหมด, จำนวนซ้ำ)

เคล็ดลับ:
- ใช้หลายเงื่อนไขพร้อมกันได้
- กด "รีเซ็ต" เพื่อล้างฟิลเตอร์

================================================================================
3. การย้ายข้อมูล
================================================================================

ขั้นตอน:
1. เปิดแท็บ "ย้ายข้อมูล"
2. เลือก Table ต้นทาง
3. เลือกชุดข้อมูลที่ต้องการย้าย (Checkbox)
4. เลือก Table ปลายทาง
5. เลือกโหมด:
   - ไม่เลือก "ลบต้นทาง" = คัดลอก
   - เลือก "ลบต้นทาง" = ย้าย
6. กด "ย้ายข้อมูล"
7. รอ Progress Bar

กรณีการใช้งาน:
- จัดระเบียบข้อมูล (ย้ายข้อมูลเก่า)
- แยกตามประเภท (Lead → Customer)
- รวมข้อมูล (คัดลอกจากหลาย Table)
- Backup (คัดลอกไปสำรอง)

คำเตือน:
- การย้ายไม่สามารถยกเลิกได้
- แนะนำคัดลอกก่อน ตรวจสอบแล้วค่อยย้าย

================================================================================
4. การส่งออกข้อมูล
================================================================================

ขั้นตอน:
1. เปิดแท็บ "ส่งออกข้อมูล"
2. เลือก Table
3. เลือกชุดข้อมูล (Checkbox)
4. ระบุจำนวนที่ต้องการส่งออก
5. (Optional) สุ่มแทรกเบอร์จากไฟล์อื่น:
   - เลือก Checkbox "สุ่มแทรก..."
   - เลือกไฟล์เบอร์เพิ่มเติม
6. กด "ส่งออก"
7. เลือกที่บันทึกและตั้งชื่อไฟล์

ระบบติดตาม:
- เบอร์ใหม่: is_exported = 0
- เบอร์ที่ส่งออกแล้ว: is_exported = 1
- ครั้งถัดไป "นำออกได้" จะลดลง

หมายเหตุ:
- หากระบุจำนวนมากกว่า "นำออกได้" จะนำเบอร์เก่ามาด้วย
- แนะนำตั้งชื่อไฟล์: "2024-11-12_Lead_Facebook.txt"

================================================================================
5. การรวม 2 ไฟล์
================================================================================

ขั้นตอน:
1. เปิดแท็บ "รวม 2 ไฟล์"
2. กดปุ่ม "เลือกไฟล์ที่ 1" → เลือกไฟล์แรก
3. กดปุ่ม "เลือกไฟล์ที่ 2" → เลือกไฟล์ที่สอง
4. กด "รวมและบันทึก"
5. เลือกที่บันทึกและตั้งชื่อไฟล์ใหม่

ระบบจะทำ:
- อ่านทั้ง 2 ไฟล์
- รวมข้อมูล
- สุ่มเรียงลำดับ (Shuffle)
- บันทึกเป็นไฟล์ใหม่

กรณีใช้งาน:
- รวม Lead จากหลายแหล่ง
- เพิ่มเบอร์เข้าไฟล์เดิม
- สุ่มลำดับก่อนใช้

หมายเหตุ:
- ไฟล์ต้นฉบับไม่ถูกแก้ไข
- บรรทัดว่างจะถูกลบอัตโนมัติ
- เบอร์ซ้ำระหว่าง 2 ไฟล์จะคงไว้ทั้งหมด

================================================================================
6. การกรองเบอร์ซ้ำ
================================================================================

ขั้นตอน:
1. เปิดแท็บ "กรองเบอร์ซ้ำ"
2. เลือก Table
3. กด "แสดงเบอร์ซ้ำ"
4. ดูผลลัพธ์:
   - เบอร์ที่ซ้ำ
   - จำนวนการซ้ำ
5. กด "ส่งออกเป็น CSV"
6. เลือกที่บันทึกไฟล์

ผลลัพธ์ CSV:
เบอร์ที่ซ้ำ    | จำนวนการซ้ำ
0812345678     | 5
0823456789     | 3
0834567890     | 2

ประโยชน์:
- ตรวจสอบคุณภาพข้อมูล
- หาต้นตอปัญหา
- ทำความสะอาดข้อมูล
- รายงานสถิติ

หมายเหตุ:
- เรียงตามจำนวนการซ้ำ (มาก → น้อย)
- CSV รองรับภาษาไทย (UTF-8-sig)
- เปิดได้ใน Excel, Google Sheets
